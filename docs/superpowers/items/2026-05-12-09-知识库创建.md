# 功能项：知识库创建

**来源:** [roadmap](../../docs/superpowers/plans/2026-04-29-ai-question-agent.md)
**创建:** 2026-05-12
**状态:** 完成

## 拆解思路

当前后端已有 knowledge/ 模块（知识点提取）和 chat.db（LangGraph checkpoint），但无知识库数据模型、API 或前端页面。最小可运行步骤：定义 KB 数据模型 + SQLite 表 + 创建/列表 API。端到端原型优先：创建知识库 → 前端可见 → 聊天中可选择，用最简配置（名称+描述）打通全链路，再逐步增强配置项和清理临时代码。

## 功能项

### F-1: 知识库数据模型与 SQLite 存储
- [x] **状态:** 完成
- **生命周期:** 交付
- **目标:** 建立 KB 数据模型和 SQLite 表，提供 CRUD 函数
- **验证:**
  - `question_agent/kb/` 模块包含 models.py（KnowledgeBase pydantic 模型）和 database.py（建表 + create_kb / list_kbs 函数）
  - 调用 create_kb 后 list_kbs 能返回已创建的记录
- **依赖:** 无
- **代码量:** ~120 行

### F-2: 知识库创建与列表 API
- [x] **状态:** 完成
- **生命周期:** 交付
- **目标:** FastAPI 路由提供 POST/GET /api/v1/knowledge-bases 端点
- **验证:**
  - POST /api/v1/knowledge-bases 返回 201 + 创建的 KB 对象
  - GET /api/v1/knowledge-bases 返回 KB 列表
  - 422 校验：名称为空时拒绝创建
- **依赖:** F-1 的数据模型和 CRUD 函数
- **代码量:** ~100 行

### F-3: 端到端知识库创建原型
- [x] **状态:** 完成
- **生命周期:** 原型
- **目标:** 前端知识库管理页面：创建知识库 + 列表展示，最简配置（名称+描述）
- **验证:**
  - /knowledge-bases 页面可创建知识库并在列表中看到
  - 创建成功后列表自动刷新
- **反馈:** 完成后暂停，演示创建流程并评估：交互是否直观？是否需要学科/学段等配置项？列表信息是否充分？
- **依赖:** F-2 的 API 端点
- **代码量:** ~180 行

### F-4: 知识库配置项增强
- [x] **状态:** 完成
- **生命周期:** 交付
- **目标:** 为 KB 添加 subject（学科）、grade_level（学段）配置字段，更新模型、API 和前端表单
- **验证:**
  - 创建 KB 时可指定学科和学段
  - 列表展示学科和学段标签
  - GET /api/v1/knowledge-bases 返回完整字段
- **依赖:** F-3 的前端页面和表单
- **代码量:** ~120 行

### F-5: 聊天中知识库选择
- [x] **状态:** 完成
- **生命周期:** 交付

- **依赖:** F-2 的列表 API
- **代码量:** ~120 行
