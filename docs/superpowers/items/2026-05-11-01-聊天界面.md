# 功能项：聊天界面 (ChatGPT 风格)

**来源:** [roadmap](../../docs/superpowers/plans/2026-04-29-ai-question-agent.md)
**创建:** 2026-05-11
**状态:** 进行中

## 拆解思路

当前前端已有 Vite + React + TanStack Router 基建和 shadcn/ui 基础组件（Button/Card/Badge），但无聊天相关代码。后端已有 FastAPI 出题 API，无 WebSocket 端点。拆解策略：先搭 ChatGPT 风格布局骨架，再用 mock 数据验证消息交互，然后通过原型打通前后端 WebSocket 通信，最后加上会话管理和流式展示。每一步系统都可运行。

## 功能项

### F-1: 聊天页面布局骨架
- [x] **状态:** 完成
- **生命周期:** 交付
- **目标:** 搭建 ChatGPT 风格双栏布局 — 左侧对话列表侧边栏 + 右侧聊天主区域
- **验证:**
  - 页面渲染为左侧边栏（对话列表 + 新建对话按钮）+ 右侧主区域（消息区 + 输入框）
  - 侧边栏可折叠，移动端默认收起
  - 使用 shadcn/ui Card/Button 组件
- **依赖:** 无（基于现有 Vite + TanStack Router 基建）
- **代码量:** ~120 行

### F-2: 消息气泡与输入交互
- [x] **状态:** 完成
- **生命周期:** 交付
- **目标:** 实现用户/助手消息气泡展示和文本输入交互，mock 回复验证布局效果
- **验证:**
  - 用户消息右对齐、助手消息左对齐，助手消息支持 Markdown 渲染
  - 输入框 Enter 发送、Shift+Enter 换行、发送按钮可点击
  - 发送后 mock 助手回复显示在消息列表中
- **依赖:** F-1 的布局骨架
- **代码量:** ~130 行

### F-3: WebSocket 端到端通信原型
- [x] **状态:** 完成
- **生命周期:** 原型
- **目标:** 打通前后端 WebSocket 通信链路 — 后端 echo 端点 + 前端 WebSocket hook
- **验证:**
  - 前端发送消息通过 WebSocket 到达后端，后端 echo 返回后前端显示
  - 连接断开时界面显示断开状态提示
- **反馈:** 🔄 完成后暂停，验证 WebSocket 通信链路稳定性和消息延迟是否可接受
- **依赖:** F-2 的消息交互
- **代码量:** ~150 行（前端 hook ~60 行 + 后端端点 ~90 行）

### F-4: 对话会话管理
- [ ] **状态:** 待开始
- **生命周期:** 过渡
- **目标:** 新建对话、对话列表展示、切换对话，使用 React state 暂存（后续由持久化层替换）
- **验证:**
  - 点击"新建对话"创建空白会话，侧边栏列表更新
  - 点击侧边栏对话项切换到对应会话，消息列表正确显示
  - 不同对话的消息互不干扰
- **依赖:** F-1 的侧边栏布局
- **代码量:** ~120 行

### F-5: 流式响应展示
- [ ] **状态:** 待开始
- **生命周期:** 交付
- **目标:** 助手消息 token-by-token 逐字渲染，模拟打字效果
- **验证:**
  - 后端分 chunk 发送响应，前端逐 token 拼接渲染
  - 响应过程中自动滚动到最新消息
  - 响应开始前显示 typing indicator
- **依赖:** F-3 的 WebSocket 通信
- **代码量:** ~100 行

### F-6: Mock 清理与样式打磨
- [ ] **状态:** 待开始
- **生命周期:** 重构
- **目标:** 移除 F-2 的 mock 回复逻辑，确保所有数据流经 WebSocket；打磨视觉细节
- **验证:**
  - 无 mock 回复代码残留，所有消息来自 WebSocket
  - 空对话状态、加载状态、错误状态均有占位提示
  - 整体视觉接近 ChatGPT 风格（间距、字体、配色）
- **依赖:** F-3, F-4, F-5
- **代码量:** ~80 行
