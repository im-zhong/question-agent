## 拆解思路

当前 ReAct 循环完全依赖 LLM 自主决定是否调工具和调哪个工具，不稳定——可能不调、调错顺序。需要添加 dispatch 节点做意图分类，对明确意图（出题、知识抽取）直接强制调工具，对自由对话走正常 ReAct。dispatch 节点通过合成 AIMessage（含 tool_calls）路由到 ToolNode，复用现有工具执行机制，无需手写工具调用逻辑。chat_node 始终负责最终的自然语言响应。

## 功能项

### F-1: 意图分类逻辑
- [x] **状态:** 已完成
- **生命周期:** 交付
- **目标:** 从用户消息中检测意图：extract（长文本/教材内容）、generate（"出题"/"生成题"）、chat（其他）
- **验证:**
  - 长文本（>200字）→ "extract"
  - 含"出题/生成题/练习题"关键词 → "generate"
  - 其他消息 → "chat"
- **依赖:** 无
- **代码量:** ~50 行

### F-2: Dispatch 节点 + 强制工具调用机制
- [x] **状态:** 已完成
- **生命周期:** 交付
- **目标:** 添加 dispatch_node，对 extract/generate 意图合成含 tool_calls 的 AIMessage，路由到 ToolNode
- **验证:**
  - dispatch 检测到 extract 意图 → 生成含 extract_knowledge_points tool_call 的 AIMessage

- **依赖:** F-1
