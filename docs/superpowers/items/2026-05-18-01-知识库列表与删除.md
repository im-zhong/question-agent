# 功能项：知识库列表与删除

**来源:** [roadmap](../plans/2026-04-29-ai-question-agent.md)
**创建:** 2026-05-18
**状态:** 完成

## 拆解思路

知识库创建（上轮）已提供 POST + GET /v1/knowledge-bases 和前端页面。核心缺口：无删除能力，前端无删除按钮。后端 delete_kb 数据库函数已存在（级联删除 knowledge_points → documents → knowledge_bases + 磁盘文件），但 DELETE 端点已暴露且前端已有删除 UI。本项验证完整性并补全测试覆盖。

## F-N 拆解

### F-1: 后端 DELETE 端点 ✅ 已存在
- [x] DELETE /v1/knowledge-bases/{kb_id} → 204
- [x] 404 for nonexistent KB
- [x] cascade: knowledge_points → documents → knowledge_bases
- [x] 磁盘文件清理 (shutil.rmtree)

### F-2: 前端删除 UI ✅ 已存在
- [x] Trash2 按钮图标
- [x] window.confirm 确认对话框
- [x] deletingKbId loading 状态
- [x] 删除后刷新列表、收起展开

### F-3: 列表端点 ✅ 已存在
- [x] GET /v1/knowledge-bases → list[KnowledgeBase]
- [x] 按 created_at DESC 排序

## 验证标准

- [x] DELETE /v1/knowledge-bases/{id} 返回 204，KB 从列表消失
- [x] DELETE 不存在的 KB 返回 404
- [x] 删除 KB 级联删除其 documents 和 knowledge_points
- [x] 删除 KB 清理磁盘文件
- [x] 连续删除同一 KB，第二次返回 404
- [x] 删除一个 KB 不影响其他 KB
- [x] 前端有删除按钮 + 确认对话框
