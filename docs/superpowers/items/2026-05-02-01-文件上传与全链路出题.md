# 功能项：文件上传与全链路出题

**来源:** [roadmap](../../plans/2026-04-29-ai-question-agent.md)
**创建:** 2026-05-02
**状态:** 完成

## 拆解思路

当前后端已有 `/questions/generate/from-file` 端点，一步完成 提取→章节→知识点→出题 全流程，CORS 已开放。
前端从零开始——最小可运行步骤是初始化 Next.js 项目并连接后端 /health 验证通信。
然后构建文件上传 UI → 调用 from-file 端点 → 展示结果，形成端到端闭环。
先用简化展示跑通全链路（原型），再用结构化组件替换原始展示，最后打磨样式和错误处理。

## 功能项

### F-1: 出题页面骨架
- [x] **状态:** 完成
- **生命周期:** 交付
- **目标:** 初始化 Next.js 项目，搭建单页布局，验证与后端 /health 的连接
- **验证:**
  - `npm run dev` 启动后浏览器访问 localhost:3000 可见页面标题和上传区域占位
  - 页面加载时调用 GET /health 返回 200，显示"后端已连接"
- **依赖:** 无
- **代码量:** ~80 行

### F-2: 文件上传交互
- [x] **状态:** 完成
- **生命周期:** 交付
- **目标:** 实现拖拽+点击文件上传组件，校验文件类型
- **验证:**
  - 拖拽或点击可选择 PDF/DOCX/TXT 文件
  - 选择非支持类型文件时显示"不支持的文件格式"提示
  - 选中文件后显示文件名和大小
- **依赖:** F-1
- **代码量:** ~120 行

### F-3: 端到端出题原型
- [x] **状态:** 完成
- **生命周期:** 原型
- **目标:** 上传文件 → 调用 POST /questions/generate/from-file → 简化展示生成的题目
- **验证:**
  - 上传教材文件后点击"生成题目"，调用 from-file 端点返回 200
  - 页面显示生成的题目列表（题干 + 选项文字，简化排版）
  - API 调用期间显示加载状态
- **反馈:** 完成后暂停，演示出题效果并评估：端到端流程是否顺畅？出题延迟是否可接受？简化展示是否满足基本可读性？
- **依赖:** F-2
- **代码量:** ~150 行

### F-4: 结构化结果展示
- [x] **状态:** 完成
- **生命周期:** 交付
- **目标:** 用卡片和折叠面板展示章节、知识点、题目和生成统计
- **验证:**
  - 章节区域可折叠展开，显示章节层级
  - 知识点列表显示类别标签（concept/formula/procedure 等）
  - 题目卡片显示题干 + A/B/C/D 选项 + 类别徽标
  - 底部显示生成统计（total/successful/failed）
- **依赖:** F-3
- **代码量:** ~200 行

### F-5: 错误处理与空状态
- [x] **状态:** 完成
- **生命周期:** 交付
- **目标:** 完善错误提示、空状态引导和失败题目的标记
- **验证:**
  - 后端不可达时显示"无法连接后端服务"提示
  - 文件上传/生成失败时显示具体错误信息
  - 失败题目（status=failed）卡片带红色边框和"生成失败"标签
  - 初始状态显示引导文案"上传教材文件开始出题"
- **依赖:** F-4
- **代码量:** ~120 行

### F-6: 样式打磨与清理
- [x] **状态:** 完成
- **生命周期:** 重构

  - 所有 shadcn/ui 组件使用统一主题色
