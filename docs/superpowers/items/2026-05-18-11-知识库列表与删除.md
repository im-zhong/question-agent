# 功能项：知识库列表与删除

**来源:** [roadmap](../plans/2026-04-29-ai-question-agent.md)
**创建:** 2026-05-18
**状态:** 完成

## 拆解思路

当前状态：KB 列表功能完整（database.py: list_kbs → router.py: GET → knowledge-bases.tsx: 卡片列表），删除功能完全缺失。外键存在但无 ON DELETE CASCADE，需手动级联删除（knowledge_points → documents → knowledge_bases → 文件目录）。已有边缘测试断言 DELETE 返回 405，添加端点后需更新。

拆解策略：先补数据库层级联删除函数，再接 API 端点，再做前端删除 UI，最后补集成测试和列表空状态。每步保持系统可运行。

## 功能项

### F-1: Database 级联删除函数
- [x] **状态:** 已完成
- **生命周期:** 交付
- **目标:** 在 database.py 中添加 delete_kb 函数，手动级联删除 knowledge_points → documents → knowledge_bases 行，并清理磁盘文件目录
- **验证:**
  - delete_kb(kb_id) 删除 KB 行及其所有 documents 和 knowledge_points 子行
  - 磁盘目录 `data/kb/{kb_id}/` 及其中文件被递归删除
  - KB 不存在时不报错（幂等）
  - 文件清理失败时记录 warning 但不阻塞数据库删除
- **依赖:** 无
- **代码量:** ~80 行

### F-2: DELETE API 端点 + 边缘测试更新
- [x] **状态:** 已完成
- **生命周期:** 交付
- **目标:** 添加 DELETE /v1/knowledge-bases/{kb_id} 端点，返回 204；更新现有断言 DELETE 返回 405 的测试
- **验证:**
  - DELETE /v1/knowledge-bases/{kb_id} 已存在 KB → 204
  - DELETE /v1/knowledge-bases/{kb_id} 不存在 KB → 404
  - test_edge_kb_api.py 中 DELETE 405 断言（集合级 DELETE 返回 405，正确）
- **依赖:** F-1
- **代码量:** ~80 行

### F-3: 前端删除按钮 + 确认对话框
- [x] **状态:** 已完成
- **生命周期:** 交付
- **目标:** 每个 KB 卡片添加删除按钮（Trash2 图标），点击弹出 AlertDialog 确认，确认后调用 DELETE API 并刷新列表
- **验证:**
  - 每个 KB 卡片右上角显示删除图标按钮
  - 点击弹出确认对话框，显示 KB 名称
  - 确认后发送 DELETE 请求，成功后列表刷新（卡片消失）
  - 删除中按钮显示加载状态
  - 删除失败显示错误 toast（Sonner）
- **依赖:** F-2
- **代码量:** ~150 行

### F-4: 删除级联集成测试
- [x] **状态:** 已完成
- **生命周期:** 交付
- **目标:** 端到端测试：创建 KB → 上传文档 → 验证 KPs → 删除 KB → 验证级联清理完成
- **验证:**
  - 创建 KB → 上传文档 → 确认 KPs 存在 → DELETE KB → GET /knowledge-bases 不含该 KB
  - 删除后 GET /{kb_id}/documents 返回 404
  - 删除后 GET /{kb_id}/knowledge-points 返回 404
  - 删除后磁盘目录不存在（unit test 覆盖）
  - tests/integration/test_walkthrough.py 覆盖 DELETE cascade 流程
- **依赖:** F-2
- **代码量:** ~100 行
