# 功能项：文档上传到知识库

**来源:** [roadmap](../plans/2026-04-29-ai-question-agent.md)
**创建:** 2026-05-13
**状态:** 完成

## 拆解思路

当前状态：KB 模块已有 create/list API + 前端页面，extraction pipeline（extractors + knowledge）已完整实现但独立运行，两者未连接。chat 发送 kb_id 但后端未使用。

拆解策略：先建数据模型（Document + KnowledgePoint），再做一个端到端原型把上传→提取→存储串通，然后补生产级状态跟踪，再做前端 UI 和聊天集成。F-2 是跨层原型（同步处理），验证全链路可行后再逐步增强。

## 功能项

### F-1: Document & KnowledgePoint 数据模型与存储
- [x] **状态:** 已完成
- **生命周期:** 交付
- **目标:** 为知识库添加文档和知识点的数据模型与 aiosqlite 持久化存储
- **验证:**
  - documents 表含 id, kb_id, filename, format, file_path, char_count, status, created_at 字段
  - knowledge_points 表含 id, document_id, kb_id, name, description, tags_json, confidence, method, chapter_id, created_at 字段
  - create_document / list_documents / create_knowledge_points (batch) / list_kbs_knowledge_points 函数可用
  - init_db 建两张新表，不破坏已有 knowledge_bases 表
- **依赖:** 无
- **代码量:** ~130 行

### F-2: 文档上传 + 提取处理端点
- [x] **状态:** 已完成
- **生命周期:** 原型
- **目标:** 实现 POST /v1/knowledge-bases/{kb_id}/documents 端到端上传：保存文件→调用已有 extractors + knowledge pipeline→存储 KPs→返回结果
- **验证:**
  - POST multipart 上传 PDF/DOCX/TXT 到指定 KB，返回 201 + document 记录 + 提取的 KP 数量
  - 文件保存到 data/kb/{kb_id}/ 目录
  - 提取的 KPs 写入 knowledge_points 表
  - GET /v1/knowledge-bases/{kb_id}/documents 返回文档列表
  - GET /v1/knowledge-bases/{kb_id}/knowledge-points 返回 KP 列表
  - KB 不存在时返回 404
- **反馈:** 完成后暂停，演示上传→提取→查询完整流程，评估：(1) 处理时间是否可接受？(2) KP 提取质量是否满足出题需求？(3) 同步处理是否需要改为异步？
- **依赖:** F-1
- **代码量:** ~200 行

### F-3: 处理状态跟踪与 KB 元数据更新
- [x] **状态:** 已完成
- **生命周期:** 交付
- **目标:** 为文档添加处理状态流转，提取失败不崩溃；上传成功后更新 KB 的 document_count
- **验证:**
  - Document status 字段支持 uploaded → processing → ready / failed 流转
  - 提取过程中异常时 status 标记为 failed，错误信息存入 error_message 字段
  - 文档处理完成后 KB 的 document_count 自增
  - GET documents 返回含 status 字段
- **依赖:** F-2
- **代码量:** ~100 行

### F-4: 前端文档上传与浏览
- [x] **状态:** 已完成
- **生命周期:** 交付
- **目标:** KB 管理页面添加文档上传入口，展示文档列表和知识点
- **验证:**
  - 每个 KB 卡片有上传按钮，点击触发文件选择
  - 上传后 KB 卡片 document_count 实时更新
  - 文档列表显示文件名、状态（ready/failed）、创建时间
  - KP 数量显示，可展开查看知识点名称列表
- **依赖:** F-2
- **代码量:** ~180 行

### F-5: 聊天工具 KB 集成
- [x] **状态:** 已完成
- **生命周期:** 交付
- **目标:** 聊天中携带 kb_id 时，从知识库查询 KPs 用于出题，替代需要用户上传文件的方式
- **验证:**
  - 聊天消息含 kb_id 且 KB 有 KPs 时，generate_questions 使用 KB 中的 KPs
  - 无 kb_id 或 KB 为空时回退到原有行为（用户上传文件或输入文本）
  - 端到端：选择 KB → 发送"出题"→ 基于该 KB KPs 生成题目
- **依赖:** F-1
- **代码量:** ~100 行

### F-6: 清理与校验
- [x] **状态:** 已完成
- **生命周期:** 重构
