# 功能项：基于知识点生成题干

**来源:** [roadmap](../../docs/superpowers/plans/2026-04-29-ai-question-agent.md)
**创建:** 2026-05-01
**状态:** 进行中

## 拆解思路

知识点标注（上轮）已提供 KnowledgePoint 的 name/description/tags ——每个知识点就是出题的天然输入单元。本项聚焦"基于知识点生成题干"，不涉及正确答案生成（下一 leaf）和干扰项逻辑建模（独立分支）。拆解线索：先定义题干数据模型 + API 骨架，再以原型打通"知识点→GLM-5→题干+简化选项"端到端链路获取早期反馈，然后按知识点类别优化 prompt 模板、支持批量生成、接入文件上传管线，最后模块封装。原型中选项为简化占位（非合理干扰项），后续 leaf 将替换为正式答案和逻辑建模干扰项。

## 功能项

### F-1: 题目数据模型与 API 骨架
- [x] **状态:** 完成
- **生命周期:** 交付
- **目标:** 定义 QuestionStem / QuestionOption Pydantic 模型，新增 POST /questions/generate 骨架端点，接收知识点列表返回占位响应。
- **验证:**
  - POST /questions/generate 接收 `knowledge_points: [{name, description, tags}]` JSON body，返回符合 QuestionStem schema 的占位响应
  - QuestionStem 含 id、stem_text、options(A/B/C/D)、knowledge_point_name、question_type、metadata 字段
  - /health、/extract、/structure、/knowledge 端点不受影响
- **依赖:** 无（复用 knowledge/ 公共 API 类型）
- **代码量:** ~80 行

### F-2: 端到端出题原型
- [x] **状态:** 完成
- **生命周期:** 原型
- **目标:** 实现 GLM-5 出题 Prompt，将知识点输入送入模型并解析为结构化题干+4个简化选项，打通"知识点→LLM→题目输出"全链路。
- **验证:**
  - 输入含"牛顿第二定律"知识点，GLM-5 返回 stem_text 非空且与知识点相关的题干，4 个选项格式为 A/B/C/D
  - GLM-5 不可用时返回空列表并记录 warning，不崩溃
  - 响应中每个题干的 knowledge_point_name 与输入知识点对应
- **反馈:** 🔄 完成后暂停，演示出题效果并评估：题干与知识点的相关性？题干表述清晰度和学段适配性？选项虽为简化占位，是否已能初步评估出题方向？
- **依赖:** F-1
- **代码量:** ~150 行

### F-3: 知识点类别感知生成
- [x] **状态:** 完成
- **生命周期:** 交付
- **目标:** 根据 KnowledgeTag.category 选择不同出题策略：concept→定义/理解题，formula→应用/计算题，procedure→步骤/操作题，fact→识记题，principle→分析/推导题。每类独立 prompt 模板 + 少样本示例。
- **验证:**
  - concept 类知识点（如"加速度的定义"）生成定义理解型题干，stem 含"定义""含义""是指"等语义
  - formula 类知识点（如"v = v₀ + at"）生成应用计算型题干，stem 含具体数值或变量替换场景
  - 多标签知识点按主标签（tags[0].category）选择 prompt 策略
  - 无标签知识点使用通用出题 prompt
- **依赖:** F-2
- **代码量:** ~120 行

### F-4: 多知识点批量生成
- [x] **状态:** 完成
- **生命周期:** 交付
- **目标:** 支持一次请求传入多个知识点并批量生成题干，包含生成统计（总数/成功/失败）和单条失败隔离。
- **验证:**
  - 传入 3+ 知识点，响应含对应数量的题干列表
  - 某个知识点的 GLM-5 调用失败时，该条标记为 failed 但不影响其他条目
  - 响应含 generation_stats: {total, successful, failed}
- **依赖:** F-3
- **代码量:** ~120 行

### F-5: 文件上传出题管线
- [ ] **状态:** 待开始

- **验证:**

  - 提取 0 个知识点时返回 questions=[] 且 generation_stats.total=0，不报错
  - /health、/extract、/structure、/knowledge、/questions/generate 端点不受影响
  - walkthrough 测试覆盖 POST /questions/generate/from-file
- **依赖:** F-4
