# 01 - 项目概览与架构

## 项目定位

question-agent-v1 是一个**智能出题 Agent**——用户上传教材文档或输入学习内容后，系统能自动：

1. 解析文档结构（章节检测）
2. 提取核心知识点
3. 根据知识点自动生成各类题目（选择题、简答题、论述题）

这不是一个简单的"调 LLM API 生成题目"的工具，而是一个**多阶段管线 + 对话式 Agent** 的复合系统，核心设计思想是：

- **规则 + LLM 混合检测**——规则保证可靠性和零成本，LLM 提升语义理解能力
- **结构化知识提取**——不是让 LLM 直接出题，而是先提取知识点，再基于知识点出题
- **对话式交互**——通过 LangGraph StateGraph 构建可中断、可恢复的对话 Agent

## 技术栈

### 后端（Python 3.12+）

| 层面 | 技术 | 用途 |
|------|------|------|
| Web 框架 | **FastAPI** | REST API + WebSocket |
| Agent 框架 | **LangGraph** | 对话式 Agent 的 StateGraph 编排 |
| LLM SDK (管线) | **zai-sdk** (智谱AI) | 章节/知识点检测的 LLM 调用 |
| LLM SDK (Agent) | **langchain-openai** | 对话 Agent 的 LLM 调用（通过 OpenAI 兼容接口） |
| 数据库 | **SQLite** (aiosqlite) | 知识库存储 + 对话状态持久化 |
| 数据校验 | **Pydantic v2** | 配置管理、请求/响应模型 |
| 文档解析 | **pdfplumber** / **python-docx** | PDF 和 DOCX 文件文本提取 |
| 代码质量 | **ruff** / **mypy** (strict) | lint + 类型检查 |

### 前端

| 层面 | 技术 | 用途 |
|------|------|------|
| 框架 | **React 19** + **TypeScript** | UI |
| 构建 | **Vite 8** | 开发/构建工具 |
| 路由 | **TanStack Router** | 文件路由 |
| 样式 | **Tailwind CSS 4** + **shadcn/ui** | 组件库 |
| 状态管理 | React Context + useState | 无外部状态库 |

### LLM 模型

使用智谱 AI 的 **GLM** 系列模型（通过 `.env` 配置 `GLM_MODEL`），默认 `glm-5-flash`。所有 LLM 调用使用 `response_format={"type": "json_object"}` 强制结构化输出。

## 双 LLM SDK 设计

本项目使用了两个不同的 LLM SDK，这是一个容易困惑的点：

| SDK | 用途 | 调用方式 | 原因 |
|-----|------|----------|------|
| **zai-sdk** | 管线中的章节/知识点检测 | `client.chat.completions.create()` | 智谱官方 SDK，直接调用 |
| **langchain-openai** | 对话 Agent | `ChatOpenAI.bind_tools()` | LangGraph 生态要求使用 LangChain 接口 |

两者连接的是同一个智谱 AI API 端点（`https://open.bigmodel.cn/api/paas/v4/`），使用同一个 API Key，只是调用方式不同。

- **zai-sdk** 用于管线中独立的一次性 LLM 调用，简单直接
- **langchain-openai** 用于 Agent 中的工具绑定和流式响应，需要与 LangGraph 集成
