# 03 - 混合检测引擎

混合检测（Hybrid Detection）是本系统最核心的设计模式，同时应用于**章节检测**和**知识点提取**两个阶段。

## 设计哲学

为什么不直接用 LLM？为什么不只用规则？

| 方案 | 优点 | 缺点 |
|------|------|------|
| 纯规则 | 零成本、确定性高、速度快 | 覆盖面有限，无法理解语义 |
| 纯 LLM | 覆盖面广、语义理解强 | 有成本、可能失败、延迟高 |
| **混合** | **规则兜底保证可用性，LLM 补充提升质量** | **合并逻辑需要仔细设计** |

核心原则：

1. **规则永远执行**——即使 LLM 不可用，系统仍然能工作
2. **LLM 作为增强**——检测规则遗漏的语义内容
3. **置信度仲裁**——当两者冲突时，信任更高置信度的结果
4. **优雅降级**——LLM 失败时自动回退到纯规则模式

## 章节检测

章节检测识别文档中的标题/章节结构，判断每个段落的层级（章/节/小节）。

### 规则检测器 (`chapters/detector.py`)

三种策略，按置信度从高到低：

```
策略 1: DOCX 样式检测 (confidence = 0.95)
  匹配 "Heading 1" / "Heading 2" / "Heading 3" 样式
  → 最可靠，因为样式是文档作者明确标注的

策略 2: 编号模式匹配 (confidence = 0.9)
  正则匹配：
    "1.1.1" → level 3
    "1.1"   → level 2
    "1."    → level 1
    "第一章" → level 1
    "第二节" → level 2
    "（三）" → level 1
    "一、"   → level 1
    "§2.3"  → level 2
  → 规范文档的编号格式非常可靠

策略 3: 字体启发式 (confidence = 0.6)
  条件: font_size >= 16 AND is_bold
  → 最低置信度，因为大号加粗不一定是标题
  → 有去重机制：连续的同级字体检测只保留第一个
```

合并策略：**同一行号（line_index），保留最高置信度的检测结果**。

### LLM 检测器 (`chapters/llm.py`)

调用 GLM-5 进行语义章节检测：

```
输入: 段落列表（含行号和文本）
Prompt: "识别文档中的章节标题，返回 JSON"
输出: [{line_index, level, title, confidence}]

特点:
  - 可以识别没有明确编号或格式的标题
  - 例如：一个段落虽然没有加粗、没有编号，但语义上是章节标题
  - confidence 固定 0.8（低于规则检测的 0.9+）
```

### 混合合并 (`chapters/hybrid.py`)

```
合并策略:
  1. 规则检测结果分为高置信度 (>= 0.8) 和低置信度 (< 0.8)
  2. 高置信度规则结果全部保留
  3. LLM 结果用于:
     - 替换低置信度规则结果（同一行号）
     - 填充规则完全没有检测到的行号
  4. 同一行号取最高置信度
  5. LLM 不可用时，返回纯规则结果（detection_method = "rule_only"）
```

```
示例:
  规则检测: 行5 "第一章 绪论" (pattern, 0.9), 行10 "1.1 背景" (pattern, 0.9)
  LLM 检测: 行5 "第一章 绪论" (0.8), 行15 "研究意义" (0.8), 行20 "1.2 目标" (0.8)

  合并结果:
    行5: 规则 0.9 > LLM 0.8 → 保留规则
    行10: 仅规则 → 保留
    行15: 仅 LLM → 补充 LLM
    行20: 仅 LLM → 补充 LLM

  detection_method = "hybrid", rule_count = 2, llm_count = 2
```

## 知识点提取

知识点提取从文本中识别核心知识单元，每个知识点有一个类别标签。

### 规则检测器 (`knowledge/detector.py`)

基于标记词（marker word）匹配：

```
类别和对应的标记词模式:

concept (概念):
  "定义是", "是指", "称为", "叫做", "指的是", "又称", "也叫", "简称为"

formula (公式):
  "公式是", "定理是", "定律是", "方程是", "公式：", "定理："

procedure (步骤):
  "第一步", "第二步", "步骤一", "步骤是"

fact (事实):
  "事实是", "数据是", "约为", "大约是"

principle (原理):
  "原理是", "规律是", "机制是"

confidence: 固定 0.7
```

特点：
- 每行每个类别最多匹配一次（避免重复）
- 知识点名称从匹配位置前后截取（前 15 字 + 后 30 字）
- 截取到句子边界处，最长 60 字

### LLM 检测器 (`knowledge/llm.py`)

```
输入: ChapterWindow（章节标题 + 文本 + 行号范围）
Prompt: 知识点提取引擎，5 大类别定义 + JSON 输出格式
输出: [{name, description, tags[{value, category}], source_line_start, source_line_end}]

参数:
  temperature = 0.1  (极低随机性，保证稳定输出)
  max_tokens = 3000
  response_format = {"type": "json_object"}
  confidence = 0.85  (固定，高于规则的 0.7)
```

LLM 的优势：
- 能识别隐含的知识点（没有标记词但确实是核心概念）
- 能自动分类（一个知识点可能同时属于多个类别）
- 能提取跨行的知识点

### 混合合并 (`knowledge/hybrid.py`)

合并策略比章节检测更复杂，因为需要处理**行范围重叠**：

```
去重逻辑:
  1. 计算两个知识点的行范围重叠率 = overlap_len / min(range1_len, range2_len)
  2. 如果重叠率 > 50%，认为是同一个知识点
  3. 冲突时保留置信度更高的那个
  4. 不重叠的结果全部保留

示例:
  规则: 行5 "加速度的定义" (concept, 0.7, range=[5,6))
  LLM:  行5 "加速度" (concept, 0.85, range=[5,8))
  重叠率: 1/1 = 100% > 50% → 同一知识点
  LLM 0.85 > 规则 0.7 → 保留 LLM 结果

  规则: 行20 "F=ma" (formula, 0.7, range=[20,21))
  LLM:  行20 "牛顿第二定律" (principle, 0.85, range=[20,23))
  重叠率: 1/1 = 100% > 50% → 同一知识点
  LLM 0.85 > 规则 0.7 → 保留 LLM 结果

  规则: 行30 "光速约为3×10⁸" (fact, 0.7, range=[30,31))
  LLM:  无对应结果
  → 规则结果保留
```

### 降级模式

```
LLM 可用时:
  有规则 + 有 LLM → "hybrid"
  有规则 + 无 LLM → 不可能（规则总在运行）
  无规则 + 有 LLM → "llm_only"

LLM 不可用时:
  有规则 → "rule_only"
  无规则 → "none"
```

## 窗口分割机制

文档可能很长，LLM 有上下文限制。窗口分割将长文本切分为适合 LLM 处理的片段。

```
策略:
  1. 按章节分组（每个章节独立处理）
  2. 每个章节内，沿段落边界切分
  3. 每个窗口最大 3000 字符
  4. 保持段落完整性（不在段落中间切分）

  ChapterWindow = {
    chapter_id: "ch_1",
    chapter_title: "第一章 绪论",
    text: "...",           # 窗口文本
    line_start: 5,         # 起始行号
    line_end: 45,          # 结束行号
  }
```

窗口分割的行号信息对混合合并至关重要——它允许规则检测和 LLM 检测的结果通过行范围进行对齐和去重。
