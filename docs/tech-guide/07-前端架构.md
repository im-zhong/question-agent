# 07 - 前端架构

前端是一个基于 Vite + React + TypeScript + Tailwind + shadcn/ui 的 SPA，通过 REST API 和 WebSocket 与后端交互。

## 技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| React | 19 | UI 框架 |
| TypeScript | 5.x | 类型安全 |
| Vite | 8 | 构建工具 |
| TanStack Router | 最新 | 文件路由 |
| Tailwind CSS | 4 | 样式 |
| shadcn/ui | base-nova | 组件库 |
| react-markdown | - | Markdown 渲染 |
| remark-gfm | - | GFM 扩展 |

## 路由

TanStack Router 文件路由，路由文件位于 `frontend/src/routes/`：

| 路径 | 文件 | 功能 |
|------|------|------|
| `/` | `__root.tsx` | 重定向到 `/chat` |
| `/chat` | `chat.tsx` | 对话式出题 |
| `/upload` | `upload.tsx` | 文件上传出题 |
| `/knowledge-bases` | `knowledge-bases.tsx` | 知识库管理 |

路由树自动生成在 `frontend/src/routeTree.gen.ts`，**不要手动编辑**。

## 状态管理

没有使用外部状态库（Redux / Zustand / Jotai），全部基于 React Context + useState。

### ChatProvider

核心状态管理器，位于 `frontend/src/lib/chat-context.tsx`：

```typescript
ChatProvider 管理的状态:
  - conversations: Conversation[]     // 对话列表
  - activeConversationId: string      // 当前活跃对话 ID
  - messages: Record<convId, Msg[]>   // 各对话的消息列表

核心方法:
  - createConversation()  → 创建新对话
  - selectConversation()  → 切换对话
  - addMessage()          → 添加消息
  - appendToken()         → 追加流式 token
  - addStructuredData()   → 添加知识点/题目卡片数据
```

### 持久化

对话元信息（不含消息内容）保存到 `localStorage`：

```
localStorage key: "chat-conversations"
存储: Conversation[] (仅 id, title, createdAt)
消息内容: 不持久化，重连时通过 WebSocket "history" 事件恢复
```

## WebSocket 集成

### useWebSocket Hook

位于 `frontend/src/hooks/use-websocket.ts`：

```typescript
const { status, send, connect, disconnect } = useWebSocket({
  onMessage: (event) => {
    const data = JSON.parse(event.data);
    switch (data.type) {
      case "start":   // 创建空消息
      case "token":   // 追加文本
      case "data":    // 添加结构化数据
      case "end":     // 完成消息
      case "history": // 恢复历史
      case "error":   // 显示错误
    }
  }
});
```

### 连接生命周期

```
组件挂载 → connect()
  → WebSocket 连接到 /ws/chat

发送消息 → send(JSON.stringify({
    conversation_id: activeId,
    content: text,
  }))

收到响应 → 通过 onMessage 回调更新状态

组件卸载 → disconnect()
```

### 知识库选择

当用户在对话界面选择了知识库时：

```typescript
// 发送消息时自动添加 kb 前缀
const prefix = selectedKbId ? `[kb:${selectedKbId}] ` : "";
send(JSON.stringify({
  conversation_id: activeId,
  content: prefix + text,
}));
```

## 组件架构

### Chat 页面

```
ChatPage
├── ChatSidebar          // 对话列表
│   ├── 新建对话按钮
│   └── ConversationItem[] // 对话条目
├── ChatMain             // 主聊天区
│   ├── MessageList       // 消息列表
│   │   └── MessageBubble[]  // 单条消息
│   │       ├── Markdown 渲染 (文本消息)
│   │       ├── KnowledgePointsList (知识点卡片)
│   │       └── QuestionsList (题目卡片)
│   └── ChatInput         // 输入框
│       ├── 文本输入
│       └── 知识库选择器
└── KbSelector (侧边栏)  // 知识库选择
```

### MessageBubble

消息气泡组件，根据消息类型渲染不同内容：

```
MessageBubble
├── 用户消息 → Markdown 渲染
└── AI 消息
    ├── 文本内容 → react-markdown + remark-gfm
    │              + 流式打字光标动画
    └── 结构化数据
        ├── type="knowledge_points" → KnowledgePointsList
        │   └── 每个知识点: 名称 + 描述 + 标签
        └── type="questions" → QuestionsList
            └── 每道题: 题干 + 选项 + 答案 + 解析
```

### 知识库管理页面

```
KnowledgeBasesPage
├── KnowledgeBaseList      // 知识库列表
│   └── KnowledgeBaseCard[] // 知识库卡片
└── KnowledgeBaseDetail    // 知识库详情（选中后）
    ├── 文档列表
    ├── 文档上传
    └── 知识点列表
```

## API 通信

### REST API

直接使用 `fetch()` 调用，通过 Vite 开发代理：

```typescript
// 开发环境: /api → http://localhost:8000
// 生产环境: VITE_API_URL 环境变量
const response = await fetch(`/api/v1/knowledge-bases`);
```

### WebSocket

```typescript
// 开发环境: /ws → ws://localhost:8000
// Vite 代理 /ws 路径
const ws = new WebSocket(`${wsUrl}/ws/chat`);
```

### Vite 代理配置

```typescript
// vite.config.ts
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8000',
      rewrite: (path) => path.replace(/^\/api/, ''),
    },
    '/ws': {
      target: 'ws://localhost:8000',
      ws: true,
    },
  },
}
```

## 样式体系

- 移动端：侧边栏可折叠
